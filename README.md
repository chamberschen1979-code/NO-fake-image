# 图像防伪工具 v1.0

图片 PS 篡改检测（ELA）+ 底板比对查重（EXIF/ORB）+ 问题图片库

## 功能

| 功能 | 说明 |
|------|------|
| **图片篡改识别** | 上传图片，ELA 检测篡改痕迹，返回评分/判定/置信度/热力图。可疑或篡改自动入库。 |
| **图片查重检测** | 两张图对比 或 与问题库对比，基于 EXIF 时间戳 + ORB 特征匹配判断是否同一底板。 |
| **问题图片库** | 管理可疑/篡改图片，支持批量上传、查看、删除。 |

# 1. 安装依赖
pip install -r requirements.txt

python web_app.py --port 8505

## 分数与判定标准

| 评分 | 判定 | 说明 |
|------|------|------|
| 0-9 | 正常 | 无篡改信号 |
| 10-24 | 可疑 | 存在局部修改可能 |
| ≥25 | 篡改 | 明确篡改 |

**置信度**（交叉验证）：
- **高** → ELA + 边缘方向一致性 均确认篡改
- **中** → 仅 ELA 检测到异常
- **正常** → 无异常

**查重标准**：
- EXIF：同一 XMP CreateDate / DateTimeOriginal
- ORB：内点率 > 2% → 同一底板（实测同一底板最低 13%，不同底板最高 0.2%）

## 安全说明

- 上传文件以 UUID 重命名存储，防止路径遍历

- 无数据库，数据存储在本地 JSON 文件

## 算法原理简述

### 一、图片篡改检测（ELA + 边缘方向交叉验证）

**核心思想：** JPEG 图片每保存一次就会重新压缩一次。如果一张图被 PS 过（比如抠图、改字），被修改的部分经历了"两次压缩"，而没改的部分只有"一次压缩"，两者的压缩痕迹不一样。

**检测流程：**

```
上传图片
    ↓
① 以 Q75 质量重新保存 JPEG → 得到"二次压缩版"
    ↓
② 对比原图和二次压缩版 → 计算每个 24×24 像素块的误差值
    ↓
③ z-score 离群分析 → 找出误差异常大的块（z > 3.5）
    ↓
④ 空间聚类 → 异常块是聚在一起的（PS）还是散落全图的（压缩伪影）
    ↓
⑤ 边缘方向交叉验证 → 在异常区域内检查边缘方向是否连续
    （PS 会破坏边缘的自然连续性，压缩伪影不会）
    ↓
输出: 评分(0-100) + 判定(正常/可疑/篡改) + 置信度(高/中/正常)
```

**关键指标说明：**

| 评分区间 | 判定 | 说明 |
|---------|------|------|
| 0-9 | 正常 | 无任何篡改信号 |
| 10-24 | 可疑 | 存在局部修改可能（需人工复核） |
| ≥25 | 篡改 | 明确篡改（自动入库） |

| 置信度 | 条件 | 含义 |
|-------|------|------|
| 高 | ELA 检测到篡改 + 边缘方向也在同一区域异常 | 两个独立算法都确认 → 铁证 |
| 中 | 仅 ELA 检测到异常 | 可能是 JPEG 压缩伪影，需要警惕 |
| 正常 | 无异常 | 安全 |

**为什么比纯 ELA 更准？**
- 旧算法：固定阈值 → 原图文字多了容易误报
- 新算法：z-score 自适应 + 空间聚类 → 能区分 PS 和 JPEG 伪影
- 双重验证：边缘方向一致性作为独立交叉验证 → 减少误判

---

### 二、图片查重检测（EXIF + ORB 特征匹配）

**查什么：** 判断两张图是不是**同一个物理地点**拍的（不管招牌是否换过）。比如：
- 同一张门头照片被 PS 改了字 → ✅ 识别为同一底板
- 同一个店面换了 KT 板重新拍照 → ✅ 识别为同一底板（背景特征不变）
- 完全不同的两个店面 → ❌ 不同底板

**检测流程：**

```
上传图片
    ↓
① EXIF 比对（优先）→ 检查 XMP CreateDate / DateTimeOriginal 是否一致
    ↓（EXIF 被微信抹除 或 时间戳不同时）
② ORB 特征点匹配（回退）→ 提取图片中 4000 个特征点（门框、墙角、窗户边缘等）
    ↓
③ Lowe's 比例测试 → 过滤掉"长得像但不是同一位置"的误匹配
    ↓
④ RANSAC 单应性几何验证 → 检查匹配点是否符合同一个空间变换关系
    ↓
输出: 内点率 > 2% → 同一底板
```

**ORB 特征点为什么能扛住换招牌？**

门头照里约 75% 的特征点落在**换不掉的地方**（门框、墙体纹理、地面接缝），只有约 25% 落在招牌上。即使招牌全部换掉，ORB 仍然能找到足够多的背景特征点来判断"这是同一个地方"。

| 场景 | ORB 内点率 | 判定 |
|------|-----------|------|
| 一模一样的图 | 100% | ✅ 同一底板 |
| PS 改了字 | 74% | ✅ 同一底板 |
| 换了 KT 板（模拟） | 63% | ✅ 同一底板 |
| 不同光照 + 角度 | 26% | ✅ 同一底板 |
| 完全不同的地方 | 0.13% | ❌ 不同底板 |

**阈值 2% 的安全余量：**
- 同一底板的最低实测值：13%（极端裁剪后）
- 不同底板的最高实测值：0.2%
- **阈值 2% 是不同底板最高值的 10 倍，安全可靠**

### 三、问题图片库的工作机制

- **自动入库**：图片篡改检测判定为"可疑"或"篡改"时，自动复制图片到问题库
- **手动入库**：通过"问题图片库 → 批量上传"手动添加
- **去重**：同名文件不可重复入库（按文件名唯一）
- **查重匹配**：每次上传图片，自动与问题库全量比对（EXIF → ORB）
